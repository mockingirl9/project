Описание проекта (@planner_ficl_bot): Telegram-бот, который автоматизирует планирование задач и помогает распределять время за счёт алгоритмов и напоминаний. Бот позволяет пошагово добавлять задачи в список дел. 
Пользователь вводит название задачи, выбирает категорию задачи из предложенного списка, выбирает дедлайн выполнения и длительность самой задачи, а также, по желанию, добавляет список вещей, которые нужно взять с собой или адрес мероприятия.
Список задач можно увидеть с помощью кнопки «Мои задачи», в нём уже автоматически проставлены приоритеты. Задачи можно удалить или пометить, как выполненные. Позже бот отправляет уведомление о дедлайне и оповещает пользователя, если задача просрочена.

База данных:
Таблица tasks содержит все поля задачи:
id — уникальный идентификатор
user_id — Telegram ID пользователя
title, category, deadline, duration (в минутах)
priority (1-3, где 3 — высший)
status (active/completed/overdue)
address и items_to_bring — дополнительная информация

Ключевые функции:

1. calculate_priority(deadline, category)
Умный расчёт приоритета:
Близость дедлайна: ≤0 дней → +50 баллов, ≤1 дня → +40, ≤3 → +30 и т.д.
Вес категории: Учеба/Работа (25 баллов), Здоровье (20), Финансы (15)...
Итог: ≥60 → 🔴 высокий (3), ≥35 → 🟡 средний (2), иначе 🟢 низкий (1)

2. find_free_slot(user_id, duration_minutes, deadline)
Ищет свободное время в расписании пользователя:
Получает все активные задачи с дедлайнами
Начинает поиск с ближайшего получаса (00 или 30 минут)
Проверяет только рабочие часы (8:00–22:00)
Ищет окно длительностью duration_minutes, не пересекающееся с другими задачами
Возвращает найденный слот или ближайший час, если ничего не найдено

3. estimate_duration(title, category)
Эвристическая оценка времени:
Быстрые ключевые слова («позвонить», «отправить»...) → 30 минут
Долгие ключевые слова («изучить», «разработать»...) → 120 минут
Категории «Учеба»/«Работа» → 90 минут
По умолчанию → 60 минут

Машина состояний (state machine)
user_states — словарь, хранящий текущий шаг создания задачи для каждого пользователя:

user_states[user_id] = {
    'step': 'title',  # текущий шаг (title, category, deadline...)
    'task_data': {}   # накопленные данные задачи
}

Процесс создания задачи (6 шагов):
Название → сохраняется в task_data['title']
Категория → выбор из предустановленного списка
Дедлайн
Длительность → с рекомендуемым значением от estimate_duration
Доп. информация → адрес и/или список вещей
Сохранение → расчёт приоритета, поиск свободного слота, запись в БД

Система напоминаний (фоновый поток)
def check_reminders():
    Запускается каждую минуту
    Проверяет все активные задачи с дедлайнами
Если дедлайн меньше текущего времени → помечает как 'overdue' и отправляет уведомление
Если до дедлайна 1 час (±2 минуты) → напоминание
Если до дедлайна 30 минут (±2 минуты) → срочное напоминание

Основной цикл обработки сообщений
handle_all_messages() — главный маршрутизатор:
Если текст соответствует кнопке меню → вызывается соответствующая функция
Если state['step'] не None → пользователь в процессе создания задачи → process_task_step()
Иначе → предлагает использовать меню
